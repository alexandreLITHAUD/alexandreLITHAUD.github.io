1. Introduction

Bash (Bourne Again SHell) is a command language and shell scripting language widely used in Unix-like operating systems. This guide provides an overview of basic Bash scripting concepts and best practices.

2. Basic Script Structure

  • Shebang Line:

    #!/bin/bash

  • Comments:

    # This is a comment

  • Execution Permission:

    chmod +x script.sh

  • Run Script:

    ./script.sh

3. Variables

  • Assigning Variables:

    variable_name="value"

  • Accessing Variables:

    echo $variable_name

4. Conditional Statements

  • if statement:

    if [ condition ]; then
  # commands
fi

  • elif statement:

    if [ condition ]; then
  # commands
elif [ another_condition ]; then
  # commands
fi

  • case statement:

    case $variable in
  pattern1)
    # commands
    ;;
  pattern2)
    # commands
    ;;
  *)
    # default commands
    ;;
esac

5. Loops

  • for loop:

    for item in list; do
  # commands
done

  • while loop:

    while [ condition ]; do
  # commands
done

  • until loop:

    until [ condition ]; do
  # commands
done

6. Functions

  • Function Declaration:

    function_name() {
  # commands
}

  • Function Call:

    function_name

  • Function with Parameters:

    function_with_params() {
  param1=$1
  param2=$2
  # commands
}

function_with_params arg1 arg2

7. Command Line Arguments

  • Accessing Command Line Arguments: 
    # $1, $2, ... represent the first, second, etc. arguments
echo "First argument: $1"

8. File Operations

  • Check if File Exists:

    if [ -e file ]; then
  # commands
fi

  • Read from a File:

    while IFS= read -r line; do
  echo "$line"
done < file.txt

  • Write to a File:

    echo "Hello, World!" > output.txt

9. Error Handling

  • Exit on Error:

    set -e

  • Custom Error Messages:

    die() {
  echo "$1" >&2
  exit 1
}

# Usage: die "Error message"

10. Best Practices

  • Use meaningful variable names.
  • Add comments for complex sections.
  • Use indentation for readability.
  • Test scripts on various environments.

11. Conclusion

Bash scripting is a powerful tool for automating tasks in a Unix-like environment. This guide covers fundamental concepts, but there is much more to explore. Refer to the Bash manual for detailed information and advanced features.

1. Introduction

Docker is a platform that enables developers to automate the deployment of applications inside lightweight, portable containers. Containers allow applications and their dependencies to be packaged together, ensuring consistency across different environments.

2. Installation

2.1. Docker Engine

  • Follow the official Docker Engine installation guide for your operating system.

2.2. Docker Compose

  • Install Docker Compose by following the instructions in the official documentation.

3. Getting Started

  • Verify your Docker installation by running docker --version and docker-compose --version.
  • Run a simple container: docker run hello-world.

4. Docker Commands

4.1. Images

  • List images:

    docker images

  • Pull an image:

    docker pull image_name:tag

  • Remove an image:

    docker rmi image_name:tag

4.2. Containers

  • List running containers:

    docker ps

  • List all containers (including stopped):

    docker ps -a

  • Run a container:

    docker run image_name

  • Stop a running container:

    docker stop container_id

  • Remove a container:

    docker rm container_id

4.3. Networking

  • List networks:

    docker network ls

  • Create a network:

    docker network create network_name

4.4. Volumes

  • List volumes:

    docker volume ls

  • Create a volume:

    docker volume create volume_name

5. Dockerfile

  • Create a Dockerfile to define the image configuration.
  • Example Dockerfile: 
    FROM base_image:tag

WORKDIR /app

COPY . .

RUN npm install

CMD ["npm", "start"]

6. Docker Compose

  • Define multi-container applications using a docker-compose.yml file.
  • Example docker-compose.yml: 
    version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "8080:80"
  app:
    build: .
    ports:
      - "3000:3000"

7. Docker Swarm

  • Docker Swarm allows you to create and manage a cluster of Docker nodes.
  • Initialize a Swarm: 
    docker swarm init

8. Best Practices

  • Keep images small.
  • Use multi-stage builds.
  • Optimize Dockerfile layers.
  • Use .dockerignore to exclude unnecessary files.
  • Avoid running processes as root in containers.

9. Troubleshooting

  • Check container logs: docker logs container_id.
  • Use docker exec -it container_id /bin/bash to access a container's shell.
  • Check the Docker documentation and community forums for assistance.

10. Conclusion

Docker simplifies the process of deploying and managing applications in a consistent and portable manner. This documentation provides a basic overview of Docker concepts, commands, and best practices. Explore the official Docker documentation for more in-depth information and advanced topics.
